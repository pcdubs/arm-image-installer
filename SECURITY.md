# Security Policy

## Supported Versions

We currently support the latest version of arm-image-installer available on the main branch.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in arm-image-installer, please report it responsibly.

### Reporting Process

**Use GitHub Security Advisories** to privately report security vulnerabilities:

1. Go to https://github.com/fedora-arm/arm-image-installer/security/advisories/new
2. **Do not** open a public GitHub issue for security vulnerabilities
3. Provide the following information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

### Response Timeline

- **Initial Response:** We aim to acknowledge receipt within 48 hours
- **Status Update:** We will provide a detailed response within 7 days
- **Resolution:** Timeline depends on severity and complexity

### Security Updates

Security fixes will be:
- Released as soon as possible after verification
- Documented in commit messages and release notes
- Published through GitHub releases

## Security Best Practices

When using arm-image-installer:
- Always verify the integrity of downloaded images
- Run the installer with appropriate permissions (typically requires root)
- Ensure your source images are from trusted sources
- Keep your system and dependencies up to date

## Disclosure Policy

We follow responsible disclosure practices and request that security researchers do the same. We will work with you to understand and address the issue before any public disclosure.
